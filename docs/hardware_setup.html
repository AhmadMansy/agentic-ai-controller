Agentic AI Controller

Physical status lights + approve button for AI coding agents. Works on macOS, Linux, and Windows. Everything you need to wire, upload, and run.

Overview Parts Pinout Wiring diagram Assembly Upload sketch Run bridge OS permissions Test it Troubleshooting

01What you're building

Three LEDs mirror your AI agent's state. A button and joystick let you approve permission prompts without touching the keyboard.

Red — busy

Lights when you submit a prompt. The agent is thinking / running tools.

Yellow — permission

The agent is blocked waiting for you to approve a tool call.

Green — ready

Turn is finished. Safe to walk away or fire off the next prompt.

Joystick + big button

Joystick up/down moves the selector in a permission dialog. Button = Enter = approve.

03Pinout reference

Every wire you'll place. This is the source of truth — the diagram and assembly steps below just visualize it.

Arduino pinConnects toFunction
D9 Red LED anode → 220 Ω → red LED cathode → GND rail BUSY lamp
D10 Yellow LED anode → 220 Ω → yellow LED cathode → GND rail PERMISSION lamp
D11 Green LED anode → 220 Ω → green LED cathode → GND rail READY lamp
D2 One side of the push button. Other side → GND rail. Approve button (active-low, uses internal pull-up)
D3 Joystick SW pin Joystick press = second approve
A0 Joystick VRy pin (Y-axis) Up / down navigation
5V Joystick VCC → 5V rail on the breadboard Power for the joystick
GND GND rail — joystick GND, both LED cathodes (via resistors), button's other leg Common ground
Joystick VRx is unused. You can leave it disconnected. Only the Y-axis drives up/down selection.

05Assembly (breadboard order)

Do these in order. Keep the Arduino unplugged from USB until the very end.

Power rails

  1. Run a jumper from the Arduino's 5V pin to the red (+) rail on the breadboard.
  2. Run a jumper from the Arduino's GND pin to the blue/black (−) rail on the breadboard. This becomes the GND rail everything else shares.

LEDs (× 3)

Each LED uses the same pattern — just change the digital pin per color.

  1. Put the LED across the breadboard gap. Long leg (+) on the Arduino side, short leg on the GND-rail side.
  2. Plug one end of a 220 Ω resistor into the same row as the long leg; the other end into a free column further right.
  3. Run a jumper from that free column to the Arduino pin:
    • red LED → D9
    • yellow LED → D10
    • green LED → D11
  4. Run a jumper from the LED's short leg row to the GND rail.
Resistor placement matters — polarity doesn't (for the resistor). It goes between the Arduino pin and the LED anode, not on the GND side. Either direction of the resistor itself is fine.

Approve button

  1. Seat the push button across the breadboard center gap so its 4 legs land on both sides.
  2. From one leg, run a jumper to D2.
  3. From the opposing-side leg, run a jumper to the GND rail.
No resistor needed for the button. The Arduino sketch enables the internal pull-up on D2, so the pin sits HIGH until the button pulls it LOW through GND.

Joystick

  1. 5V rail → joystick VCC
  2. GND rail → joystick GND
  3. A0 → joystick VRy
  4. D3 → joystick SW
  5. Leave VRx disconnected.

Final check before USB

  • Every LED's short leg reaches GND (not the long leg).
  • No bare wire is shorting 5V directly to GND on the rails.
  • The joystick VCC is on 5V, not 3.3V.

06Upload the sketch

  1. Plug the Arduino into your computer with the USB cable.
  2. Open the Arduino IDE, then File → Open… and choose
    arduino/agentic_ai_controller/agentic_ai_controller.ino
  3. Set Tools → Board → Arduino Uno.
  4. Set Tools → Port → /dev/cu.usbmodem* (macOS), /dev/ttyACM* (Linux), or COM* (Windows). The exact number may vary.
  5. Click the Upload arrow (→) in the toolbar. Wait for "Done uploading" at the bottom.
Expected: the moment upload finishes, all three LEDs flash briefly in sequence (red → yellow → green), then the green LED stays on. That's the boot self-test — it confirms wiring and sketch both work.

07Install & run the Python bridge

The bridge owns the serial port, serves a tiny HTTP API for agent hooks, and turns the physical button/joystick into Enter/↑/↓ keystrokes.

cd bridge
pip3 install -r requirements.txt
python3 agentic_ai_bridge.py

On startup you should see:

[bridge] opening /dev/cu.usbmodem1101 @ 9600
[bridge] http://127.0.0.1:8787/led/{red,yellow,green,off,test}
[bridge] Ctrl-C to quit

…and the LEDs cycle once on the Arduino. That means the bridge found the board.

Leave this terminal window open. Closing it shuts the bridge down and the LEDs stop reacting.

08OS permissions (one-time)

On macOS, synthetic key presses require Accessibility permission. The first time you push the physical button, macOS will pop up a dialog like:

"Terminal would like to control this computer using accessibility features."

  1. Click Open System Settings.
  2. Go to Privacy & Security → Accessibility.
  3. Toggle Terminal (or iTerm / whichever app runs the agent) on.
  4. Quit and re-run the bridge so it picks up the new permission.
If you skip this (macOS): the button still sends BTN over serial and the bridge still logs it, but nothing happens. That's the usual "the button does nothing" failure mode.

Linux & Windows: no extra permission step is needed. The bridge injects keystrokes directly via pynput. On some Linux distros you may need to add yourself to the dialout or uucp group for serial port access (see the README).

09Test the whole loop

  1. With the bridge running, open a second terminal and do a quick LED check: 
    curl http://127.0.0.1:8787/led/test
curl http://127.0.0.1:8787/led/red
curl http://127.0.0.1:8787/led/yellow
curl http://127.0.0.1:8787/led/green
curl http://127.0.0.1:8787/led/off
    Each curl should change the LEDs instantly.
  2. Start your AI coding agent in a terminal and keep that window focused.
  3. Type a prompt and hit enter. Red should come on immediately.
  4. Make the agent want to run a tool that triggers a permission prompt. Yellow lights.
  5. Push the joystick up/down — the selector should move.
  6. Press the big button — the highlighted choice should be accepted (same as pressing Enter).
  7. When the agent finishes the turn, green comes on.

10Troubleshooting

Bridge says "no serial device matching …"

  • Plug the Arduino in, then re-run the bridge.
  • Check the exact path with ls /dev/cu.usbmodem* (macOS), ls /dev/ttyACM* (Linux), or check Device Manager (Windows).
  • Arduino IDE's Serial Monitor (or another bridge instance) will hog the port. Close it, then start the bridge.

LEDs don't light at all on boot

  • The LED is backwards. Flip it — long leg on the Arduino-pin side.
  • The resistor is not in series. It must be between the Arduino pin and the LED's long leg (or between the short leg and GND — either spot works, but one of them is mandatory).
  • The cathode isn't on the GND rail — trace the wire back.

One LED works, another doesn't

  • Swap the suspect LED with a known-good one. LEDs fail often on long leads.
  • Compare wiring against the pinout table — D9 / D10 / D11 are the three you need, not D12 / D13.

Button press does nothing

  • Is the terminal running the agent the frontmost window? pynput sends keys to whichever app has focus.
  • OS permissions — see section 08 (macOS Accessibility, Linux/Windows group membership).
  • Check the bridge terminal: does [bridge] BTN -> Enter print when you press? If yes, it's an accessibility / focus issue, not wiring.
  • Check your button's legs — on a standard 4-pin tactile button, the opposing pair is the one that bridges when pressed, not adjacent legs. Try the other two legs.

Joystick scrolls by itself (drift)

  • The module's mechanical center isn't exactly at 512. Edit JOY_CENTER_LO / JOY_CENTER_HI in the .ino and re-upload — widen the window to 380 / 640.

Agent feels sluggish after adding hooks

  • The LED hook already uses async: true and a 1-second timeout, so a dead bridge can't stall the agent. If it still feels slow, run the bridge; most of the time this makes the perceived difference.

Hooks not firing at all

  • In Claude, run the /hooks command to confirm the three events show both an afplay and a curl entry.
  • If you edited ~/.claude/settings.json during a session, Claude only reloads hooks when you open /hooks or restart. Open /hooks once to force a reload.
